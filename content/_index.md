# fluidity

fluidity is an agent development environment (ADE) designed for keeping more of the software development workflow in one place.

Modern development work often sprawls across editors, terminals, browser tabs, review tools, and AI agent sessions. fluidity aims to make that work feel organized and coordinated, so you can move between streams of work without losing context.

For free (*coming soon*) and [open source](https://github.com/owenps/fluidity).

## core features

<div class="feature-grid">
  <article class="feature-card">
    <div class="feature-icon feature-icon-branch" aria-hidden="true"></div>
    <h3>native git worktrees</h3>
    <p>Every feature, bugfix, or experiment gets its own isolated worktree. No stashing, no branch juggling.</p>
  </article>

  <article class="feature-card">
    <div class="feature-icon feature-icon-menu" aria-hidden="true"></div>
    <h3>tile workspaces</h3>
    <p>Build your workspace out of focused tiles: code, terminals, git diffs, browsers, issues, PRs, and agent sessions.</p>
  </article>

  <article class="feature-card">
    <div class="feature-icon feature-icon-monitor" aria-hidden="true"></div>
    <h3>keyboard-first</h3>
    <p>Move fast without reaching for the mouse. Vim-friendly navigation, quick switching, and command-driven workflows.</p>
  </article>

  <article class="feature-card">
    <div class="feature-icon feature-icon-justify" aria-hidden="true"></div>
    <h3>integrated git review</h3>
    <p>Review diffs, inspect changes, compare branches, and manage work-in-progress without leaving the app.</p>
  </article>

  <article class="feature-card">
    <div class="feature-icon feature-icon-copy" aria-hidden="true"></div>
    <h3>reusable arrangements</h3>
    <p>Define project-specific workspace setups so common workflows open with the right tiles already in place.</p>
  </article>

  <article class="feature-card">
    <div class="feature-icon feature-icon-robot" aria-hidden="true"></div>
    <h3>multiple AI agents</h3>
    <p>Coordinate different agent sessions and providers side-by-side, choosing the right tool for each stream of work.</p>
  </article>
</div>
